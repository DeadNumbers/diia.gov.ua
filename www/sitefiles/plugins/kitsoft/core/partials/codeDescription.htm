Базові вказівки щодо написання коду для пошукових роботів з прикладами


У даному файлі зберігаються РЕКОМЕНДАЦІЇ для пошукових роботів, яких вони можуть не завжди притримуватись.

Файл ігнорується усіма пошуковими роботами якщо його розмір сягає більше ніж 30kb.

Щоб закоментувати стрічку - необхіодно на початку поставити символ #.

Можна використовувати регулярні вирази, де * - означає будь яку кількість символів, а $ -кінець стрічки. Приклад: Disallow: /*.pdf$

Особливість $ на прикладі Disallow: /item$ - забороняє індексувати директорію item але не може заборонити індексацію всіх файлів зі словом item в URL.

Завжди закривайте слешом назву директорії. У деякому розумінні / означає: все що знаходиться після директиви.


User-agent: Для якого робота пишеться файл
User-agent: * - для усіх роботів
User-agent: GoogleBot - для визначеного. (Загалом їх 302)

Disallow/: Заборонити індексацію у визначеному місці
Disallow: / - заборонити індексацію по всьому сайту
Disallow: /admin/ - заборонити індексацію у директорії admin
Disallow: /file.php - заборонити індексувати файл file.php

Disallow - це обов'язкова директива. НЕ використовувати її неможна.


Allow: - дозволити індексацію за певним шляхом чи у файлі(ах) у забороненій директорії
Allow: /folder/ - дозволити індексувати директорію folder
Allow: /folder/*.js - дозволити індексувати усі JS-файли у директорії folder. (Корисно коли: Disallow: /folder/)

Sitemap: - адреса до sitemap.xml
Sitemap: site.com/sitemap.xml

Вказується без протоколів http/https.

Вказується у будь якому місці без прив'язки для конкретного user-agent.

Можна вказати декілька правил Sitemap.


Crawl-delay: - мінімальна затримка між індексацією сторінок в секундах
Crawl-delay: 10 - затримка між сусідніми завантаженнями сторінок для індексації - 10 секунд
Crawl-delay: 0.5 - затримка між сусідніми завантаженнями сторінок для індексації - пів секунди

Директива використовується для зменшення навантаження на сервер.

Стандартна затримка складає 2 секунди.

Підтримується Yandex, Mail.Ru, Bing, Yahoo.


Request-rate: - кількість сторінок не більше якої завантажувати за визначений проміжок часу
Request-rate: 1/5 - завантажувати не більше 1 сторінки за 5 секунд

Аналог Crawl-delay тільки для "західних" роботів.


Crawl-delay: 3 РІВНОЗНАЧНО Request-rate: 1/3


Visit-time: - час у який проводити індексацію
Visit-time: 0600-0845 - завантажувати сторінки тільки у проміжку 06:00 - 08:45 за Грінвічем

Clean-param: - повідомляє які сторінки не індексувати за наявністю певного параметру в URL
Clean-param: show /folder/get_file.php - не індексувати сторінку за посиланням www.site.com/folder/get_file.php?show=some&id=1

Ця директива використовується тільки для Yandex.

Yandex рекомендує використовувати цю директиву для того, щоб не враховувати UTM-мітки та індентифікатори сесій. Приклад:

Clean-Param: utm_source&utm_medium&utm_campaign

Host: - вказати основний домен сайту серед дзеркал, що буде для пошукової системи головним
Host: site.com - вказати головним дзеркалом site.com для видачі пошуковими системами
Host: www.site.com - вказати головним дзеркалом www.site.com для видачі пошуковими системами

Якщо сайт працює за протоколом HTTP його вказувати ЗАБОРОНЕНО.

Host: http://www.site.com - НЕправильна форма запису

Якщо сайт працює за протоколом HTTPS його вказувати ОБОВ'ЯЗКОВО.

Host: https://www.site.com - правильна форма запису

Директива вказується лише раз.

Це не обов'язкова директива, вона використовується тільки для Yandex.

Дана директива зазвичай вказується вкінці файлу.


GoogleBot не враховує директиви Host і Crawl-delay.

GoogleBot не може отримати доступ до файлів CSS та JS. Їх необхідно спеціально дозволи за допомогою директиви Allow.

YandexNews не враховує Crawl-delay.

YandexDirect і YandexVideoParser не враховують загальні директиви (User-agent: *), лише спеціальні для них.
